브라우저 기반으로 위치 요청을 보내서, DB에 3초마다 현재 위치의 경도/위도 데이터를 저장한다.

이렇게 해서, MapLibre/CARTO Basemap 기반으로 불러온 지도 위에
DB에 있는 데이터를 기반으로 점을 찍어서 표시해 준다.

PC로 테스트 해 봤을때 현재 위치가 생각보다 정확하게 찍힌다. 크게 벗어나지 않고 정확한 지점을 표시한다.
PC에 GPS가 있거나 하진 않을 수 있어서 브라우저가 자동으로 위치를 네트워크 기반, GPS 기반 등 알아서 찾는 것 같은데,

모바일로 외부에서 LTE/5G 데이터+브라우저 위치요청 으로 얼마나 정확하게 찍힐 지.. 나중에 산책 나가면 실험해 봐야겠다.

그리고 모바일이라고 해도, 앱 형태로 백그라운드 작동 허용을 해줘야 화면을 꺼도 잘 수집할 것 같은데,

현재는 브라우저 기반이라 아마 화면이 켜져 있는 상태, 브라우저가 화면에 떠 있는 상태에서만 위치 수집이 동작할 것으로 보인다.

이 부분 때문에 실제로는 사용하기 좀 불편할 수 있지만, 혼자서 로그 찍고 맵 만들어나가기는 어느정도 타협하면 가능할 것 같다.

그리고 회원가입/로그인으로 유저 데이터 분리가 꼭 필요할 거고, DB에 저장할 때 ID/PW뿐만 아니라 위치 데이터도 암호화하여

사람이 직접 봐도 육안으로 식별할 수 없는 형태로 암호화하여 DB에 저장해도록 해 놓아야 한다.
